# mt5-pnl-ui — README screenshot and polish

Date: 2026-07-06. Status: approved.

A review of README.md against the [Art of
README](https://github.com/hackergrrl/art-of-readme) guidelines. The
README already follows the guide's shape — name, one-liner, cognitive
funnelling from broad to specific, brevity, upfront caveats, linked
background — so this is a polish pass, not a rewrite. Four changes:
add a generated screenshot, add two missing links, and remove two
elements that do not earn their space.

## Goals

- Let a visitor evaluate the project visually in seconds: one
  screenshot of the Overview view near the top of the README.
- Make the screenshot reproducible and safe: generated by a script
  from the synthetic e2e fixture, so real trading data can never
  appear in it.
- Fix the guide's "necessary background context with links" gaps:
  `mt5-pnl-exporter` and the age encryption format are named but never
  linked.
- Tighten the README by removing the Contents table and the static
  Pages badge.

## Non-goals

- Light/dark screenshot variants. The app has a single theme, so one
  image suffices.
- Screenshots of Calendar, Trades, or Strategies. One image keeps the
  README lean; the live app is one click away.
- CI enforcement of screenshot freshness. Refreshing is a manual step
  after visible UI changes.
- Any other README restructuring.

## Screenshot generation

A new Playwright spec, `e2e/screenshot.spec.ts`, run via a new npm
script:

```bash
npm run screenshot   # regenerates docs/screenshot.png
```

- **Mechanism**: reuses the existing e2e machinery — the production
  build served by `vite preview` (via the Playwright `webServer`
  config), the real worker, and the encrypted synthetic fixture
  `e2e/fixtures/snapshot.json.gz.age` with its `e2e-passphrase`.
  Synthetic data is guaranteed by construction.
- **Capture**: load the fixture, wait for the Overview charts to
  render, then `page.screenshot()` at a 1280×800 viewport with
  `deviceScaleFactor: 2`, written to `docs/screenshot.png`
  (overwriting the previous image).
- **Separation from the e2e suite**: `playwright.config.ts` gains two
  projects — the default e2e project, which ignores
  `screenshot.spec.ts`, and a `screenshot` project matching only that
  file. `npm run e2e` runs the e2e project only; `npm run screenshot`
  runs the screenshot project. The existing suite's behaviour is
  unchanged.
- **Refresh policy**: manual. After a visible UI change, rerun
  `npm run screenshot` and commit the updated image alongside the
  change.

## README changes

- **Screenshot**: embedded directly under the one-liner blockquote,
  before the ASCII architecture diagram — funnel order: what it is,
  what it looks like, how it fits together. Descriptive alt text
  summarises what the image shows, so no information is image-only.
- **Links**: `mt5-pnl-exporter` links to
  `https://github.com/tanem/mt5-pnl-exporter` on first mention; the
  `.age` suffix links to `https://age-encryption.org` on first
  mention.
- **Removals**: the Contents section (GitHub's auto-generated header
  outline covers navigation at this length) and the Pages badge (its
  value is hard-coded and reflects nothing; the live URL remains
  prominent in "Use it"). The licence and CI badges stay.

- CLAUDE.md's Commands block gains `npm run screenshot`.
- CLAUDE.md gains a one-line note that `docs/screenshot.png` is
  generated by that script and should be refreshed after visible UI

## Testing

- The screenshot script is verified by running it once and inspecting
  the output image.
- `npm run e2e` must pass unchanged, confirming the new project split
  does not affect the existing suite.
- No unit tests: the spec exercises the same path the e2e suite
  already covers.
